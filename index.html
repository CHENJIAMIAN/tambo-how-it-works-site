TAMBO · HOW IT WORKS

让 Agent 直接说你的 UI 语言

核心机制：你先注册组件并声明 Zod Schema，Tambo 把它们变成 LLM 可调用的工具，Agent 在对话中按需调用，然后把 props 流式推送到前端组件渲染。

STEP 01

注册组件

把 Graph、TaskBoard、Note 这类 React 组件注册给 Tambo。

→
STEP 02

Schema 变工具

propsSchema（Zod）会自动转换为 LLM tool definitions。

→
STEP 03

Agent 调用

用户说“按区域展示销售”，Agent 决定调用 Graph 并生成 props。

→
STEP 04

流式渲染

组件按流式 props 渲染，支持状态更新、取消、重连与恢复。

const components: TamboComponent[] = [{
  name: "Graph",
  description: "Displays data as charts",
  component: Graph,
  propsSchema: z.object({
    data: z.array(z.object({ name: z.string(), value: z.number() })),
    type: z.enum(["line", "bar", "pie"]),
  }),
}];

Why this matters

  • 不用手动维护“工具输出 → 组件映射”
  • 同一个会话里支持可交互组件持续更新
  • MCP、Local Tools、上下文注入都可并入链路
